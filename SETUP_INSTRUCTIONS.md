# Инструкции по подключению к GitHub

Для отправки этого репозитория на GitHub выполните следующие шаги:

## 1. Создайте репозиторий на GitHub
- Зайдите на https://github.com
- Нажмите кнопку "New repository"
- Введите имя репозитория (например, "holst")
- Выберите тип (public или private)
- НЕ ставьте галочку "Initialize this repository with a README"
- НЕ добавляйте .gitignore или license (мы уже создали .gitignore)
- Нажмите "Create repository"

## 2. Свяжите локальный репозиторий с удаленным
После создания репозитория на GitHub выполните в терминале:

```bash
git remote add origin https://github.com/ВАШ_АККАУНТ/holst.git
```

Замените "ВАШ_АККАУНТ" на ваш никнейм на GitHub и "holst" на имя вашего репозитория.

## 3. Отправьте код на GitHub
```bash
git branch -M main
git push -u origin main
```

## 4. Дополнительные рекомендации
- Если вы используете токены для аутентификации, вам будет предложено ввести токен вместо пароля
- Убедитесь, что у вас установлены Git Credentials Manager или настроена аутентификация

## Статус репозитория
Ваш локальный репозиторий готов к отправке на GitHub:
- ✓ Инициализирован Git
- ✓ Настроен .gitignore
- ✓ Выполнен начальный коммит

## 5. Работа с репозиторием (push/pull)

### Отправка изменений на GitHub (push)
После внесения изменений в код выполните следующие команды:

```bash
# Проверьте статус файлов
git status

# Добавьте измененные файлы к коммиту
git add .

# Сделайте коммит с описанием изменений
git commit -m "Описание ваших изменений"

# Отправьте изменения на удаленный репозиторий
git push
```

### Получение изменений с GitHub (pull)
Чтобы получить последние изменения из удаленного репозитория:

```bash
# Получите последние изменения из удаленного репозитория
git pull

# Альтернативно, можно использовать отдельные команды:
git fetch  # Загружает изменения, но не объединяет их
git merge  # Объединяет загруженные изменения с текущей веткой
```

### Работа с ветками
```bash
# Создайте новую ветку для новой функциональности
git checkout -b feature/название-функции

# Переключитесь на существующую ветку
git checkout имя-ветки

# Объедините ветки (например, в main)
git checkout main
git merge feature/название-ветки

# Удалите ветку после объединения
git branch -d feature/название-ветки
```

### Решение конфликтов
Если при выполнении `git pull` возникают конфликты:
1. Git покажет файлы с конфликтами
2. Откройте эти файлы и найдите маркеры конфликта `<<<<<<<`, `=======`, `>>>>>>>`
3. Решите конфликты вручную
4. Добавьте файлы обратно в коммит: `git add .`
5. Завершите объединение: `git commit`

### Полезные команды
```bash
# Проверьте историю коммитов
git log --oneline

# Отмените локальные изменения в файлах
git checkout -- имя-файла

# Отмените последний коммит (но сохраните изменения)
git reset --soft HEAD~1

# Полностью удалите последний коммит изменения
git reset --hard HEAD~1
```
